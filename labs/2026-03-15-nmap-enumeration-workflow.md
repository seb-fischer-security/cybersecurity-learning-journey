# 🔎 Lab 03 – Nmap Enumeration Workflow

Date: Mar 15, 2026 | Author: S. Fischer | Tool: nmap on Kali Linux

---

## 🎯 Objectives
- Perform an advanced Nmap enumeration scan
- Combine service detection, default scripts and OS detection
- Identify additional information exposed by router services
- Document findings for security assessment

---

## 🔍 Target Overview

| Property | Value |
|--------|------|
| IP | 192.168.1.1 |
| Services | 53/tcp, 80/tcp, 443/tcp |
| Device | Home Router (ZTE firmware) |

---

## 📊 Scan Results

### 1️⃣ Enumeration Scan (-sC -sV -O)

Command:

```
nmap -sC -sV -O -oA lab03_router_enum 192.168.1.1
```

Duration: ~35 seconds

Purpose of flags:

| Flag | Description |
|------|-------------|
| -sC | Run default NSE scripts |
| -sV | Detect service versions |
| -O | Attempt OS detection |
| -oA | Save results in multiple formats |

Generated output files:

```
lab03_router_enum.nmap
lab03_router_enum.xml
lab03_router_enum.gnmap
```

---

### 📡 Detected Services

| Port | State | Service | Version |
|-----|------|--------|--------|
| 53/tcp | open | DNS | Unknown banner |
| 80/tcp | open | HTTP | ZTE web server 1.0 (2015) |
| 443/tcp | open | HTTPS | ZTE web server (SSL enabled) |

---

### 🌐 HTTP Enumeration Findings

The NSE scripts revealed additional information from the router web interface:

```
Server: ZTE web server 1.0 ZTE corp 2015
Set-Cookie: SID=...
X-Frame-Options: SAMEORIGIN
Content-Security-Policy: frame-ancestors 'self'
X-XSS-Protection: 1; mode=block
```

Key observations:

- Router redirects HTTP → HTTPS
- Session cookies are used for authentication
- Basic security headers are enabled
- Web interface appears to be custom ZTE firmware

---

### 🔐 SSL Certificate Enumeration

```
Subject: CN=192.168.1.1
Organization: ZTE
Country: CN
Valid from: 2016
Valid until: 2031
```

This confirms the router uses a **self-signed certificate generated by the firmware**.

---

### 🖥️ OS Detection

Nmap attempted operating system detection but results were inconclusive:

```
Warning: OSScan results may be unreliable
```

Possible device classifications suggested:

- VoIP adapter
- network bridge
- embedded router device

This behaviour is common for consumer routers due to **limited fingerprinting responses**.

## 🔒 Security Assessment

- ✅ **Limited attack surface** – only essential router services exposed
- ⚠️ **ZTE web server 1.0 (2015)** detected on ports 80 and 443
- ⚠️ **DNS banner exposure** may reveal firmware information
- ✅ **Security headers present** (CSP, XSS protection, frame restrictions)
- ℹ️ **Self-signed SSL certificate** typical for router web interfaces

---

## 📌 Key Takeaways

- Combining multiple Nmap options provides deeper enumeration results
- NSE scripts reveal useful service metadata beyond simple port scans
- Router web interfaces often expose identifiable firmware information
- Saving scan output (`-oA`) is useful for documentation and later analysis
